## ✅ Sistema de Checkpoints

A lo largo del curso encontrarás **checkpoints** (casillas de verificación) distribuidas en los documentos. Estos son tu forma de confirmar que entiendes cada sección.

### ¿Qué son los Checkpoints?

Los checkpoints son listas de verificación (*checklists*) que demuestran tu comprensión de cada tema. **No están solo al final**—están distribuidas estratégicamente para asegurar que leas y comprendas TODO.

### Cómo Marcar un Checkpoint

**Sin completar** (así aparecen):
```markdown
- [x] Item que no has entendido
```

**Completado** (cómo debe quedar):
```markdown
- [x] Item que ya comprendiste
```

Simplemente reemplaza el espacio con una `x`: `[ ]` → `[x]`

### El Flujo Completo

```
Lees 01-INSTALACION → Marcas sus 3 checkpoints → git push
                           ↓
Lees 02-QUE-ES-GIT → Marcas sus 3 checkpoints → git push
                           ↓
Lees 03-POR-QUE-GIT → Marcas sus 3 checkpoints → git push
                           ↓
Lees 04-CONECTAR-GITHUB → Marcas sus 3 checkpoints → git push
                           ↓
Lees 05-COMANDOS → Marcas sus 3 checkpoints → git push
                           ↓
Lees 06-GITHUB → Marcas sus 3 checkpoints → git push
                           ↓
Creas ALUMNO.md → Marcas sus 2 checkpoints → git push
                           ↓
        GitHub califica automáticamente
                           ↓
    Tu profesor asigna calificación final
```

### Calificación Automática

- **Parte Automática:** GitHub cuenta cuántos checkpoints marcaste (20 total)
  - 20/20 checkpoints = 100/100 automático ✅
  - Si faltan checkpoints, la puntuación baja proporcionalmente
  - Si faltan checkpoints, la puntuación baja

- **Parte Manual:** Tu profesor evalúa tu archivo `ALUMNO.md` (+ 50 puntos)
  - Verifica tu reflexión y aprendizaje
  - Calificación final = promedio de ambas partes

### Errores Comunes

**❌ "Marqué checkpoints pero no veo calificación"**
- Olvidaste hacer `git push`. Sin push, GitHub no ve los cambios.

**❌ "Mi calificación es 0/100"**
- Los archivos no están en `/docs/` o tienen nombres diferentes
- Verifica: `01-INSTALACION.md`, `02-QUE-ES-GIT.md`, etc.

**❌ "Marqué checkpoints sin leer"**
- Tu profesor lo verá en tu ALUMNO.md. Sé honesto en tu reflexión.

### ✅ Consejos Finales sobre Checkpoints

- ✅ Lee TODO el documento antes de marcar checkpoints
- ✅ Marca TODOS los checkpoints (31 en total)
- ✅ Haz commits después de cada documento (`git add docs/` + `git commit` + `git push`)
- ✅ Sé honesto en tu ALUMNO.md (refleja lo que realmente aprendiste)
